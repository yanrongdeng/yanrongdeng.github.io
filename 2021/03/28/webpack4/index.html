webpack 的简介

什么是 webpack？

概念： webpack 是前端项目工程化的具体解决方案。

什么是工程化？

  1. 模块化：js 的模块化、css 的模块化、资源的模块化
  2. 组件化：复用现有的UI结构、样式、行为
  3. 规范化：目录结构的划分、编码规范化、接口规范化、文档规范化、Git分支管理
  4. 自动化：自动化构建、自动部署、自动化测试

webpack 的主要功能

它提供了友好的前端模块化开发支持，以及代码压缩混淆、处理浏览器JavaScript的兼容性、性能优化等强大的功能。

webpack 的好处

让程序员把工作的重心放在具体功能的实现上，提高了前端开发效率和项目的可维护性。

安装 webpack

  1. 全局安装 webpack： npm install webpack webpack-cli webpack-dev-server -g
  2. 项目下安装 webpack(常用)： npm install webpack webpack-cli webpack-dev-server --save-dev

项目目录结构

  1. 新增项目目录： mkdir project
  2. 进入 project 文件： cd project
  3. 新建两个项目结构： mkdir src config
  4. 初始化本地仓库，生成 .git 文件夹： git init .
  5. 生成依赖文件 packgae.json ： npm init -y
  6. 生成 index.html 和 index.js 文件： touch src/index.js src/html/index.html
  7. 打开 vscode 代码工具： code .

配置 webpack

  1. 入口文件： config/webpack.config.js

  2. 配置 webpack.config.js 文件：

    entry(入口) 可以有一个，也可以有多个 string | Array
    mode(执行环境) 告知 webpack 使用相应环境的内置优化 开发(development)&生产(production)
    output(出口) 有且只有一个 object

    context(上下文)
    plugins(插件) Array<new 构造函数> 插件是 webpack 的支柱功能 插件目的在于解决 loader 无法实现的其他事 可以为loader带来更多特性
    devServer(本地服务) object
    devtool source-map是一个信息文件，里面储存着位置信息
    targets(构建目标)
    resolve(解析) resolver 是一个库(library)，用于帮助找到模块的绝对路径。一个模块可以作为另一个模块的依赖模块，然后被后者引用
    optimization(优化)
    performance(性能)
    externals(外部扩展)
    manifest runtime 在浏览器运行过程中，webpack用来连接模块应用程序所需的所有代码

    包括： 模块交互时,连接模块所需的加载和解析逻辑
    已经加载到浏览器的连接模块逻辑
    未加载模块的延迟加载逻辑

    module(模块) module { rules:[] } loader 加载器

    特性 ： 1、支持链式传递 ，反顺序执行
    2、可以是同步的，也可以是异步的
    3、可以运行在 node.js 中，可以执行任何 node.js 能做到的操作
    4、可以利用 options 配置
    5、用 module.rules 来引用模块
    6、loader 能够产生额外的任意文件
    插件的使用 和 创建方式 loader 用于转换某些类型的模块，而插件则可以用于执行范围更广的任务。
    包括：打包优化，资源管理，注入环境变量。

打包命令

命令：webpack --config ./config/webpack.config.js
发布上线打包(压缩)：webpack --config ./config/webpack.config.js --mode production

–mode 优先级高于 webpack.config.js 中的 mode

启动服务器

命令：webpack-dev-server --config ./config/webpack.config.js

webpack-dev-server 监听项目源代码的改变，只要保存了修改，webpack会自动进行项目的打包和构建
(放在虚拟内存中，而不是磁盘中，因为内存的访问速度快)
使项目运行在 http://127.0.0.1:8080
在 webpack.config.js 中设置 devServer

loader 加载器的使用

webpack处理不了css，less，image文件以及高级JS语法，需要loader帮忙处理

loader 调用的过程

  1. webpack 默认只能打包处理 .js 结尾的文件，处理不了其它后缀的文件
  2. 由于代码中包含了 main.css 这个文件，因此 webpack 默认处理不了
  3. 当 webpack 发现某个文件处理不了的时候，会查找 webpack.config.js 这个配置文件， 看 module.rules 数组中， 是否配置了对应的 loader 加载器。
  4. webpack 把 main.css 这个文件，先转交给最后一个 loader 进行处理（先转交给 css-loader)
  5. 当 css-loader 处理完毕之后， 会把处理的结果，转交给下一个 loader (转交给 style-loader)
  6. 当 style-loader 处理完毕之后， 发现没有下一个 loader 了，于是就把处理的结果，转交给了 webpack
  7. webpack 把 style-loader 处理的结果，合并到 /dist/bundle.js 中，最终生成打包好的文件。
